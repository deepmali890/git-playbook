# Public vs Private Repositories

## Public Repository

A repository that is visible to everyone on the internet.

### Features

* Anyone can view the code
* Anyone can fork the repository
* Supports open-source collaboration
* Searchable on GitHub and search engines

### Best For

* Open-source projects
* Learning projects
* Portfolio projects
* Libraries and tools

### Example

```text
Portfolio Website
Java DSA Practice
Open Source Library
```

---

## Private Repository

A repository that is only accessible to authorized users.

### Features

* Restricted access
* Better security
* Protects sensitive code
* Full control over permissions

### Best For

* Personal projects
* Commercial projects
* Client work
* Projects containing secrets or API keys

### Example

```text
Company Project
Client Website
Internal Tools
```

---

## Public vs Private

| Public                     | Private                          |
| -------------------------- | -------------------------------- |
| Visible to everyone        | Visible to authorized users only |
| Better for showcasing work | Better for sensitive projects    |
| Encourages collaboration   | Provides stronger privacy        |
| Good for portfolios        | Good for business projects       |

---

## How to Choose?

Use **Public** when:

* You want to showcase your work
* You want community contributions
* The code is safe to share

Use **Private** when:

* The project contains sensitive data
* It is client or company work
* The code should not be publicly accessible

---

## Real-World Examples

### Public

* Personal Portfolio
* Git Playbook
* Open Source Projects

### Private

* E-commerce Client Project
* Company ERP System
* Internal Business Tools

---

## Quick Notes

* Public = Open to everyone
* Private = Restricted access
* Portfolio projects → Public
* Client/company projects → Private
